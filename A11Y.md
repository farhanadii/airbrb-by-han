# Accessibility Documentation

## Overview
Accessibility features implemented using Material-UI components and React best practices.

## Implemented Features

### Semantic HTML
- Proper heading hierarchy (h1-h6 via Typography component)
- Semantic nav, main, header elements
- Button elements for all actions
- Form elements with proper structure

### Material-UI Built-in Accessibility
Material-UI components provide accessibility features out of the box:
- **TextField**: Automatic label association with inputs
- **Button**: Proper button roles and keyboard interaction
- **Dialog**: Focus management and escape key handling
- **IconButton**: Accessible button elements
- **Alert**: Screen reader announcements for messages
- **Chip**: Proper semantic structure

### Keyboard Navigation
- Tab navigation through all interactive elements
- Enter key support for form submission (implemented via handleKeyPress)
- Escape key closes modals (Material-UI Dialog default)
- Material-UI components handle focus order automatically

### Form Accessibility
- All TextField components have visible labels
- Error messages displayed in Alert components
- Required fields indicated
- Form validation with clear feedback

### Image Accessibility
- Alt text provided for all images:
  - Listing thumbnails: `alt={listing.title}`
  - Gallery images: `alt={title - Image ${index}}`
  - Property images: Descriptive alt text

### Color and Contrast
- High contrast text colors:
  - Primary text: #222 (dark charcoal) on white background
  - Secondary text: #717171 (medium gray)
- Status colors with sufficient contrast:
  - Success/Accepted: #10b981 (green)
  - Pending/Warning: #f59e0b (yellow/amber)
  - Error/Declined: #ef4444 (red)
- Color not sole indicator: Status also shown with text labels

### Responsive Design
- Mobile-first approach
- Touch targets meet 44x44px minimum
- Content reflows on different screen sizes
- Responsive typography (xs, sm, md breakpoints)
- Grid layouts adapt: 1 column mobile, 2-4 columns desktop

### Focus Management
- Material-UI components provide visible focus indicators
- Dialog components trap focus when open
- Focus returns to trigger element on modal close

### Screen Reader Support
- Proper heading structure for page sections
- Alert components announce messages
- Loading states communicated via text
- Form errors shown in Alert components

### Interactive Feedback
- Loading states displayed as text
- Error and success messages in Alert components
- Form validation with clear messaging
- Disabled button states when form invalid

## Compliance Level

Current implementation provides:
- **Basic accessibility**: Yes
- **Keyboard navigation**: Partial
- **Screen reader support**: Basic via Material-UI
- **WCAG 2.1 Level A**: Mostly compliant
- **WCAG 2.1 Level AA**: Partially compliant
