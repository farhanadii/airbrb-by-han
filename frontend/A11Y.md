# Accessibility Documentation

## Overview
Accessibility features implemented to meet WCAG 2.1 Level AA standards.

## Key Features

### Semantic HTML
- Proper heading hierarchy (h1-h4)
- Semantic elements (nav, main, section)
- Form labels associated with inputs
- Button elements for actions

### Keyboard Navigation
- All elements focusable via Tab
- Enter for form submission
- Escape to close modals
- Logical tab order
- Visible focus indicators

### ARIA Attributes
- aria-label for icon buttons
- aria-labelledby for dialogs
- aria-describedby for field descriptions
- aria-invalid for validation errors
- role attributes where needed

### Color and Contrast
- WCAG AA compliant contrast ratios (4.5:1)
- Primary text: #222 on white
- Error states: High contrast red
- Success states: High contrast green
- Information not conveyed by color alone

### Focus Management
- Visible focus indicators
- Focus trapped in modals
- Focus restored on modal close

### Form Accessibility
- All inputs have labels
- Error messages associated with fields
- Required fields marked
- Helpful placeholder text

### Image Accessibility
- Descriptive alt text for images
- Empty alt for decorative images
- Icons with text labels or aria-labels

### Responsive Design
- Mobile-first approach
- Touch targets minimum 44x44px
- Content reflows on zoom
- Text readable at 200% zoom

### Screen Reader Support
- Proper heading structure
- Status messages via aria-live
- Form errors announced
- Loading states communicated

### Motion and Animation
- Reduced motion support
- Animations can be disabled
- No auto-playing videos
- Smooth transitions

## Testing
- Keyboard navigation testing
- Screen reader testing (NVDA, JAWS, VoiceOver)
- Color contrast checking
- Automated testing (axe DevTools, Lighthouse)
- Zoom testing at 200%
- Mobile screen reader testing
